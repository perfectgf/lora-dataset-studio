# 使用应用

三种数据集类型共用同一条主流程：准备图片、筛选、生成标注、训练并在测试工作室比较结果。区别在于哪些内容必须从标注中省略。

## 角色数据集流程（参考图 → 训练好的 LoRA）

1. 新建数据集，填写名称和触发词，类型选择“角色”。
2. 上传一张清晰的正脸参考图；需要时使用裁剪调整。
3. 在“添加图片”中生成不同角度和构图的变体，也可以导入真实照片。
4. 在“图片”中保留合格图片、拒绝错误身份或明显瑕疵。
5. 在“筛选与清理”中检查人脸相似度、水印和小图修复结果。
6. 在“标注”中生成描述场景、服装、姿势和构图的文本，但不要描述角色身份。
7. 导出训练 ZIP，或在“训练”中启动本地/云端任务。
8. 在“检查点与 LoRA”和“测试工作室”中比较轮次并保存优选设置。

角色 LoRA 的身份应绑定到触发词，因此头发、肤色、脸型等永久身份特征不应反复写进标注。

## 概念数据集（对象或动作，而不是人物）

新建数据集时选择“概念”，并明确填写“概念描述”。图片应覆盖不同人物、环境、角度和光线，但都包含同一个要学习的概念。

标注需要描述画面的其余内容，却省略概念本身。若概念是特定姿势或效果，不要在每条标注中直接写出该姿势或效果，否则模型会把它绑定到普通文本而不是触发词。

概念数据集通常通过导入或网页采集构建，不需要角色参考图和人脸相似度流程。

## 风格数据集（全局审美）

风格数据集没有激活触发词；加载 LoRA 后会直接生效，并通过 LoRA 权重控制影响强度。

图片应在主体、场景和构图上保持多样，只共享同一种视觉语言。标注只描述画面内容，不写艺术家、媒介或审美名称，避免模型把风格误学成特定物体。

## 节省训练次数的技巧

- 先决定目标模型系列，再准备分辨率和标注风格。
- 优先保留身份准确、构图清楚的图片，不用数量掩盖质量问题。
- 避免大量近似图片；角度、距离、服装和背景需要有变化。
- 训练前处理全部待定图片，并检查缺失标注和泄漏提示。
- 先运行较短的测试训练，再根据测试工作室结果调整数据集。
- 保存有效的训练预设和测试优选设置，方便复现。
