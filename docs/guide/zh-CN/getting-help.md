# 获取帮助

遇到卡住、缺陷或功能缺失时，可以使用两个反馈渠道：

- **Discord**——[discord.gg/j6hnJBFtXE](https://discord.gg/j6hnJBFtXE)，通常适合快速排查；功能建议请发到 `#roadmap`。
- **GitHub**——[Issues](https://github.com/perfectgf/lora-dataset-studio/issues)，适合可以稳定复现的缺陷和功能请求。

## 怎样让问题报告更容易解决

一次有效报告通常包含四项：

1. **版本**——“设置 → 维护 → 更新”中显示的当前构建。
2. **环境**——操作系统，以及使用仅 API、完整本地还是 Docker。
3. **操作、预期和实际结果**——分别用简短句子说明。
4. **日志**——复制服务器日志末尾最接近错误的部分。

请不要粘贴 API 密钥、访问令牌或完整个人目录中的敏感内容。

## 让应用自动整理报告

下方的“诊断报告”按钮会汇总版本、操作系统、能力状态、非敏感设置和最近日志，并复制为适合粘贴到 Discord 或 GitHub 的文本。

报告不会包含已保存的密钥，但发送前仍建议快速检查一次。

## 功能请求

请说明要解决的实际工作流问题，而不只是按钮名称。最好补充：

- 当前需要经过哪些步骤；
- 哪一步最耗时或最容易出错；
- 预期的最小改进；
- 是否有可参考的应用、截图或文件格式。

## 支持项目

最有效的支持方式是提交可复现的问题、补充文档、测试新版本，或在 Discord 分享可靠的训练经验和兼容配置。
