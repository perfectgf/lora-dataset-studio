# 故障排查

先复制“设置 → 维护”中的诊断报告和服务器日志末尾，再根据下面的症状检查。

## 测试工作室或训练面板提示“没有可用的 Z-Image 模型”

确认 ComfyUI 安装目录正确，并且模型文件位于应用扫描的检查点目录。保存设置后重新检查，再刷新能力状态。使用自定义权重时，路径必须指向真实模型文件。

## 全新安装提示“找不到 SDXL 检查点”

应用不会自动附带 SDXL 检查点。将兼容模型放入 ComfyUI 的 `models/checkpoints`，或在训练设置中选择可用的官方/云端基础模型。

## 参考图裁剪没有对准脸部

自动头部裁剪依赖 Ollama 视觉模型。确认 Ollama 正在运行且视觉模型就绪；也可以关闭自动裁剪，上传后使用剪刀按钮手动调整。

## 训练日志数分钟没有变化

首次运行可能在下载模型、建立缓存或预处理图片。查看服务器日志和任务页是否仍有磁盘、网络或 GPU 活动。若超过设置的停滞超时且步数没有变化，云端守护会终止任务。

## 无法检测 ai-toolkit（conda / uv / 没有 venv）

目录应指向包含 `run.py` 的 ai-toolkit 根目录。若没有 `venv` 或 `.venv`，在“设置 → 本地工具”中填写明确的 Python 解释器路径，然后测试。

## Reddit 扫描提示限流并要求等待（429）

这是 Reddit 的频率限制。等待提示的秒数后重试，减少连续搜索，并尽量限定 Subreddit。配置 Reddit 凭据可提高稳定性，但仍需遵守平台限制。

## ComfyUI 显示无法访问

确认 ComfyUI 正在运行、API 地址正确且端口没有被防火墙阻止。默认地址通常为 `http://127.0.0.1:8188`。便携版的安装目录应指向内层 `ComfyUI` 文件夹。

## Klein 引擎一直是灰色

Klein 需要 ComfyUI 可访问，并且模型、文本编码器和 VAE 都是有效文件。设置向导会列出缺少或损坏的资源；一致性 LoRA 是推荐项，但不阻止就绪。

## macOS 上端口 5000 冲突

macOS 可能由 AirPlay Receiver 占用 5000。应用默认使用 5050；若仍冲突，请在服务器设置中更换端口并保存重启。

## Windows 控制台出现乱码

使用 UTF-8 终端，或在启动前运行 `chcp 65001`。网页界面不受控制台代码页影响；日志文件仍以 UTF-8 保存。

## `npm install` 提示缺少 `@rollup/rollup-<platform>-...`

删除前端目录中的 `node_modules` 和锁文件后重新运行 `npm install`。确认 Node.js 版本受支持，并避免在不同操作系统之间复用同一个 `node_modules`。

## 云端训练任务似乎卡住

在“训练任务”页面查看实例状态、步数和最后活动时间。启动阶段可能需要下载大型模型。不要在计费实例仍运行时只关闭浏览器；使用停止按钮，或按诊断信息登录 vast.ai 处理保留实例。
