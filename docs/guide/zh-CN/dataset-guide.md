# 构建高质量数据集

数据集决定 LoRA 学到什么。模型系列、图片构成、标注方式和训练参数必须互相匹配。

## 1. 先选择模型系列

| 系列 | 标注形式 | 常见用途 |
|---|---|---|
| Z-Image | 自然语言 | 角色、概念、风格，生成速度和质量均衡 |
| SDXL | Booru 标签 | 适合原生标签生态和成熟工作流 |
| Krea 2 | 自然语言 | 高质量审美和角色训练 |
| FLUX.1 | 自然语言 | 高提示词理解能力 |
| FLUX.2 Klein | 自然语言 | Klein 系列训练与生成 |

在开始标注前确定系列，因为更换系列可能需要重做标注风格和训练设置。

## 2. 图片数量与选择

通常 20–50 张多样化图片比大量近似图片更有效。角色数据集应覆盖脸部、半身、全身和少量背面；概念与风格数据集更需要不同主体、背景和构图。

保留图片时检查：

- 身份或目标概念准确；
- 没有明显生成错误、重复图片和严重压缩；
- 构图分布均衡；
- 水印已移除或图片已拒绝；
- 小图修复候选已做出选择。

## 3. 标注——最关键的一步

标注描述希望模型保持可控的内容；触发词承载希望 LoRA 学习的核心。

- **角色：** 描述姿势、服装、环境和构图，省略身份特征。
- **概念：** 描述场景的其余部分，省略概念本身。
- **风格：** 描述画面内容，省略审美、媒介和艺术家。

使用“泄漏检查”和“标注工具”查找反复出现的无关词。重复服装、背景或物体可能被错误绑定到触发词。

## 4. 设置速查

- **训练步数：** 数据少时不宜盲目增加；过高容易记忆训练图。
- **LoRA rank：** 越高容量越大，但也更容易过拟合并增加文件大小。
- **分辨率：** 受模型系列和显存限制；多尺度可改善不同构图适应性。
- **检查点间隔：** 保留足够中间结果，方便在测试工作室选择最佳轮次。
- **网络与优化器：** 优先使用预设；只有明确观察到问题时再调整高级参数。
- **蒙版：** 角色训练可降低背景影响，但必须确认蒙版实际生成成功。

## 5. 训练前检查

- 所有图片都已保留或拒绝，没有遗漏待定项。
- 每张保留图片都有符合数据集类型的标注。
- 泄漏检查没有未处理警告。
- 构图分布符合目标。
- 目标模型文件、训练引擎和输出目录可用。
- 云端训练已设置价格和预算限制。

## 6. 训练后选择检查点

不要默认最后一个检查点最好。将多个轮次导入测试工作室，在相同提示词、种子和强度下比较：

- 身份或概念是否稳定；
- 提示词控制是否仍然有效；
- 是否出现过拟合、固定服装或固定背景；
- 不同强度下是否平滑；
- 多种场景和构图是否都能工作。

保存优选设置后，可继续训练、部署到 ComfyUI 或导出分享。
